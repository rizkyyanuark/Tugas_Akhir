\chapter{METODE PENELITIAN}

\section{Dataset}

Penelitian ini menggunakan dataset mandiri yang dikumpulkan khusus untuk merepresentasikan aktivitas akademik domain Infokom di Universitas Negeri Surabaya (UNESA), karena ketiadaan dataset publik yang terintegrasi. Data dipilih berdasarkan relevansi dengan aktivitas dosen untuk mendukung \textit{knowledge base}, dengan setiap atribut dirancang spesifik untuk mekanisme \textit{Question-Answering}.

Dataset dihimpun dari empat sumber: (1) profil dosen \citep{pddikti_db}, (2) publikasi \textit{Google Scholar} \citep{google_scholar}, (3) data riset internal \citep{sinta_database}, dan (4) publikasi \textit{Scopus} \citep{scopus_db}. Fokus data mencakup dosen enam program studi: Sains Data (14 dosen), Kecerdasan Artifisial (5), Teknik Informatika (9), Sistem Informasi (10), Magister Informatika (5), dan Teknik Elektro (13). \section{Alur Penelitian}

Alur penelitian dirancang dengan mekanisme umpan balik (\textit{feedback loop}) untuk menjamin validitas setiap tahapannya, seperti diilustrasikan pada Gambar \ref{fig:flowchart-utama}. Proses dimulai dengan studi literatur dan pengumpulan data yang bersifat iteratif; jika data dinilai belum lengkap (Decision 2), sistem akan melakukan \textit{looping} kembali ke tahap \textit{scraping}. Demikian pula pada tahap konstruksi graf dan evaluasi sistem, validasi dilakukan secara bertingkat. Jika akurasi sistem di bawah ambang batas (Decision 5), alur akan kembali ke optimasi strategi \textit{retrieval} sebelum sistem diimplementasikan ke dalam antarmuka pengguna (\textit{GUI Implementation}).

          \begin{enumerate}[label=\textbf{\alph*.}, leftmargin=0.75cm, itemsep=5pt]
              \item \textbf{Data Identitas (PDDikti \& SimCV):} Profil dosen diambil dari PDDikti dan \textit{cv.unesa.ac.id} untuk mendapatkan identitas unik (NIP, NIDN) dan atribut demografis (Jabatan Fungsional, Prodi). Data ini membentuk node ``Aktor'' dalam graf.

              \item \textbf{Publikasi Umum (\textit{Google Scholar}):} \textit{Google Scholar} menjadi sumber utama publikasi. Teknik \textit{rotating proxy} dengan API \textit{ScraperAPI} digunakan untuk mengatasi pembatasan akses (\textit{rate limiting}) saat \textit{scraping}. Data ini (Judul, Ketua, Anggota, Sumber Dana) memetakan kolaborasi dan topik riset yang mungkin tidak tercatat dalam publikasi formal.

              \item \textbf{Publikasi Global (Scopus/SciVal):} Metadata publikasi terindeks internasional yang berafiliasi dengan UNESA diekspor dari SciVal \citep{scopus_db} sebagai pelengkap literatur global.
          \end{enumerate}

          \begin{enumerate}[label=\textbf{\alph*.}, leftmargin=0.75cm, itemsep=4pt]
              \item \textbf{Data Cleaning \& Normalization:} Memproses data mentah untuk menangani \textit{missing values} dan menstandarisasi variasi penulisan nama institusi atau singkatan gelar, langkah ini krusial untuk mencegah duplikasi entitas yang tidak perlu di dalam graf.

              \item \textbf{Text Chunking (Pemotongan Teks):} Mengingat keterbatasan \textit{context window} pada LLM, dokumen panjang seperti abstrak perlu dipecah menjadi segmen semantik yang lebih kecil (misal 512 token) dengan metode \textit{semantic chunking}. Strategi ini memastikan bahwa unit informasi yang diambil sistem tetap utuh dan spesifik, menghindari \textit{noise} dari teks yang tidak relevan.

              \item \textbf{Embedding Generation:} Mengonversi setiap \textit{chunk} teks menjadi representasi vektor numerik berdimensi tinggi menggunakan model \textit{embedding} (misal: \textit{all-MiniLM-L6-v2}). Vektor ini memungkinkan sistem melakukan pencarian berbasis makna (\textit{semantic search}) yang melampaui sekadar pencocokan kata kunci leksikal.
          \end{enumerate}

              \item \textbf{\textit{Parallel Processing Pipeline}}

                    \hspace*{0.75cm}Sistem memproses data akademik melalui dua jalur paralel, masing-masing dioptimalkan untuk karakteristik data yang berbeda. Kedua jalur berjalan secara independen namun menghasilkan output yang saling melengkapi pembangunan knowledge graph. Alur lengkap ditunjukkan pada \ref{fig:detail-kg-construction}.

                    \begin{enumerate}[label=\textbf{\alph*.}, leftmargin=0.75cm, itemsep=5pt]

                        \item \textbf{Jalur Data Terstruktur: Entity Resolution}

                              \hspace*{0.75cm}Data terstruktur (profil dosen, afiliasi) melewati \textit{Entity Resolution} untuk menyatukan referensi duplikat. Tantangan utama variasi penulisan nama (misal ``Aditya Prapanca'', ``A. Prapanca'') diselesaikan untuk menghasilkan satu \textit{canonical node} per dosen. Hal ini menjamin akumulasi publikasi yang akurat pada satu entitas peneliti.

                        \item \textbf{Jalur Data Tidak Terstruktur: NLP-Based Extraction}

                              \hspace*{0.75cm}Tahap awal \textbf{Text Processing \& Coreference Resolution} membersihkan abstrak dari \textit{noise} dan menerapkan \textit{coreference resolution} untuk mengganti kata ganti (``metode ini'', ``itu'') dengan entitas referensinya. Langkah ini krusial untuk mempertahankan konteks penuh saat pemrosesan kalimat.

                              \hspace*{0.75cm}Selanjutnya, \textbf{Ekstraksi Entitas dan Relasi} menggunakan model \textit{GLiNER (Generalist NER)} \cite{zaratiana2023gliner} dan \textit{GLiRel (Generalist Relation Extraction)} \cite{boylan2025glirelgeneralistmodel}. \textit{GLiNER} bekerja secara \textit{zero-shot} tanpa \textit{fine-tuning} spesifik, sementara \textit{GLiRel} menangani ekstraksi relasi. Kombinasi ini dipilih karena adaptabilitasnya yang tinggi untuk \textit{knowledge graph} tanpa memerlukan dataset pelatihan teranotasi besar. Output tahap ini berupa \textit{Lexical Graph} berisi triplet kandidat.

                    \end{enumerate}

              \item \textbf{\textit{Entity Linking}}

                    \hspace*{0.75cm}Tahap ini berfungsi sebagai jembatan antara kedua jalur di atas. Entitas yang diekstraksi dari teks abstrak seperti penyebutan nama dosen atau topik riset dipetakan ke entitas kanonikal yang telah divalidasi pada jalur data terstruktur. Proses \textit{Entity Linking} ini memastikan bahwa ketika sebuah abstrak menyebutkan ``Deep Learning'', sistem menghubungkannya ke node topik ``Deep Learning'' yang sudah ada di graf, bukan membuat node baru yang duplikat. Hasil dari tahap ini adalah \textit{Scholarly Knowledge Graph} yang utuh, di mana entitas terverifikasi dari jalur \textit{Structured} bersatu dengan relasi dan topik dari teks \textit{Unstructured}, tanpa duplikasi node.

              \item \textbf{\textit{Dual Indexing}}

                    \hspace*{0.75cm}Untuk mendukung \textit{Hybrid Retrieval}, hasil konstruksi KG disimpan dalam dua format:

                    \begin{enumerate}[label=\textbf{\alph*.}, leftmargin=0.75cm, itemsep=5pt]
                        \item \textbf{\textit{Vector Indexing}}

                              \hspace*{0.75cm}Teks abstrak diindeks dalam \textit{Vector Store} untuk pencarian kemiripan semantik. Teks dipotong (\textit{chunking}) (2.024 karakter, overlap 50 karakter), dilanjutkan dengan proses menyisipkan vektor \textit{embedding} menggunakan model \texttt{all-MiniLM-L6-v2} (384 dimensi), dan disimpan di \textit{Weaviate} untuk performa tinggi.

                        \item \textbf{\textit{Graph Indexing}}

                              \hspace*{0.75cm}Struktur \textit{knowledge graph} disimpan di basis data graf \textit{Neo4j}. Triplet entitas dan relasi hasil ekstraksi dimuat otomatis melalui \textit{Neo4j Python driver} atau \textit{Cypher queries} yang membuat node dan edge. Indexing ini memfasilitasi pencarian berbasis struktur relasional dan \textit{graph traversal}.

                    \end{enumerate}

          \end{enumerate}

              \item \textbf{Pengambilan Petunjuk Semantik (\textit{Clues}) pada \textit{Vector Database}}

                    \hspace*{0.75cm}Tahap ini mengidentifikasi entitas kunci dan istilah penting dari pertanyaan pengguna menggunakan pencarian berbasis vektor. Tujuannya adalah memetakan maksud pengguna ke terminologi yang konsisten dengan ontologi graf seperti misalnya, memetakan ``AI'' ke entitas topik ``Kecerdasan Buatan''

                    \hspace*{0.75cm}Pengambilan petunjuk ini diawali dengan proses pencarian kesamaan (\textit{similarity search}) pada \textit{vector database}. Sistem memanfaatkan fungsi \textit{embedding} untuk memetakan pertanyaan pengguna $Q$ ke dalam ruang vektor, kemudian mengukur kedekatannya dengan \textit{keyword} dalam korpus menggunakan metrik \textit{cosine similarity}. Proses seleksi kandidat \textit{clues} didefinisikan secara formal sebagai berikut:

                    \begin{equation}
                        K^{\text{clues}} = \{k \in K^{\text{content}} \mid \text{sim}(f_{\text{embed}}(Q), f_{\text{embed}}(k)) \geq \tau\}
                    \end{equation}

                    \hspace*{0.75cm}Setelah clues diekstraksi, Model bahasa besar (\textit{LLM}) mengubahnya menjadi dua tingkat kata kunci yang saling melengkapi:

                    \begin{equation}
                        (\mathcal{K}_h, \mathcal{K}_l) = \text{LLM}_{keyword}(Q, K^{\text{clues}})
                    \end{equation}

                    Kata kunci \textit{High-level} $\mathcal{K}_h$ menangkap konsep abstrak dan tema umum
                    penelitian (mis. ``Deep Learning'', ``Sequence Processing''), sementara kata kunci \textit{Low-level} $\mathcal{K}_l$ menangkap detail teknis spesifik (mis. ``Attention Mechanism'', ``Encoder-Decoder Models''). Strategi dua tingkat ini memastikan bahwa sistem dapat menjawab pertanyaan dengan dua jenis spektrum: pertanyaan yang memerlukan pemahaman holistik (menggunakan $\mathcal{K}_h$) dan pertanyaan yang memerlukan presisi tinggi (menggunakan $\mathcal{K}_l$).

                    \hspace*{0.75cm}Tahap ini difokuskan untuk menjawab pertanyaan yang membutuhkan ketepatan tinggi terhadap entitas tertentu. Sistem melakukan \textit{anchor node matching} pada graf untuk menemukan entitas yang relevan dengan petunjuk (\textit{clues}) yang telah diekstrak sebelumnya. Setelah node ditemukan, sistem melakukan \textit{graph traversal} (penelusuran graf) sejauh 1-hop atau 2-hop untuk mengambil konteks tetangga terdekat.

                    \hspace*{0.75cm}Untuk menjawab pertanyaan yang memerlukan pemahaman lokal, sistem menggunakan metode ekstraksi subgraf yang diarahkan oleh kata kunci \textit{low level} $K^l$. Detailnya dijelaskan sebagai berikut:

                    \begin{enumerate}[label=\textbf{\arabic*.}, leftmargin=*, itemsep=6pt]

                        \item \textbf{\textit{Node Matching} (Pencocokan Entitas):} \textit{Graph nodes} dicocokkan dengan kata kunci \textit{low level} ($\mathcal{K}_l$) menggunakan metrik \textit{vector similarity}, dengan memprioritaskan \textit{nodes} yang memiliki \textit{cosine similarity} lebih tinggi terhadap \textit{keyword embeddings}.

                        \item \textbf{\textit{Subgraph Construction}:} Setelah titik jangkar ditetapkan, sistem melakukan penelusuran jalur untuk menghubungkan simpul-simpul tersebut. Algoritma pencarian jalur terpendek (\textit{shortest path algorithms}) diterapkan untuk menemukan rute paling efisien yang menghubungkan entitas-entitas terkait.

                        \item \textbf{\textit{Pruning}:} Untuk menjaga fokus informasi, sistem mengeliminasi \textit{node} atau \textit{edge} yang dianggap \textit{outlier} atau memiliki relevansi rendah terhadap konteks utama, sehingga subgraf menjadi lebih padat dan bermakna.

                        \item \textbf{\textit{Text Unit Extraction}:} \textit{Textual chunks} yang diasosiasikan dengan \textit{retained edges} dan muncul dalam \textit{one-hop ego-graph} dari \textit{nodes} diperingkat berdasarkan frekuensi dalam subgraf, dengan segmen peringkat teratas diprioritaskan untuk \textit{retrieval}.

                        \item \textbf{\textit{Edge Expansion}:} Subgraf diperluas secara iteratif dengan menggabungkan \textit{one-hop neighbors} dari \textit{matched nodes}, memperkenalkan \textit{additional edges} yang memenuhi kriteria relevansi.

                        \item \textbf{\textit{Data Truncation}:} Untuk mematuhi batasan komputasi, seluruh data yang diekstraksi, termasuk \textit{nodes}, \textit{edges}, dan \textit{text units}, dipangkas secara bertahap hingga total jumlah token memenuhi ambang batas yang telah ditentukan.

                    \end{enumerate}

                    \hspace*{0.75cm}Rangkaian proses sebelumnya menghasilkan keluaran berupa konteks lokal terstruktur yang akan dikirimkan ke model generasi. \ref{fig:pembentukan-subgraf} menunjukan bagaimana subgraf diambil dan secara matematis, hasil akhir dari \textit{pipeline local retrieval} ini dinotasikan sebagai himpunan tiga komponen utama: entitas ($\mathcal{E}$), relasi ($\mathcal{R}$), dan unit teks ($\mathcal{T}$), sebagaimana dinyatakan dalam persamaan berikut:

                    \begin{equation}
                        \mathcal{L}_{\text{local}} = (\mathcal{E}_{\text{local}}, \mathcal{R}_{\text{local}}, \mathcal{T}_{\text{local}})
                        = \text{pipeline}_{\text{local}}(\mathcal{K}_l, G).
                    \end{equation}

                    \noindent
                    dimana persamaan ini menegaskan bahwa konteks lokal $\mathcal{L}_{\text{local}}$ adalah fungsi dari kata kunci spesifik dan struktur graf global, yang telah direduksi menjadi subset informasi yang paling signifikan untuk menjawab kueri pengguna.


              \item \textbf{Ekstraksi Jejaring Global (\textit{Global Edge Network Retrieval})}

                    \hspace*{0.75cm}Jalur ini dirancang untuk menjawab pertanyaan yang bersifat eksploratif atau membutuhkan pemahaman tematik yang luas (\textit{sensemaking}), di mana jawaban tidak bergantung pada satu entitas spesifik melainkan pada sintesis dari berbagai hubungan antar konsep. Sistem memanfaatkan Kata Kunci Tingkat Tinggi (\textit{High-level Keywords}) $\mathcal{K}_h$ sebagai basis pencarian untuk mengekstraksi \textit{Global Edge Network}. Berbeda dengan metode lokal yang menelusuri graf berdasarkan kedekatan topologi (\textit{hops}), metode ini menggunakan pendekatan berbasis vektor untuk menemukan relasi yang relevan secara semantik di seluruh graf, tanpa batasan jarak antar \textit{nodes}.

                    \hspace*{0.75cm}Proses ekstraksi informasi global ini terdiri dari rangkaian tahapan sistematis sebagai berikut:

                    \begin{enumerate}[label=\textbf{\arabic*.}, leftmargin=*, itemsep=6pt]

                        \item \textbf{\textit{Edge Retrieval}:} Basis data vektor dikueri untuk mengidentifikasi \textit{edges} yang representasi vektornya berkorelasi dengan kata kunci tingkat tinggi, dengan memprioritaskan \textit{edges} yang merangkum hubungan konseptual makro (seperti kerangka teori atau tren lintas domain).

                        \item \textbf{\textit{Nodes Retrieval}:} \textit{Nodes} yang terkait dengan \textit{edges} terpilih diekstraksi, kemudian diurutkan berdasarkan derajat keterhubungannya (\textit{degree}).

                        \item \textbf{\textit{Edge Data Processing}:} \textit{Edges} yang terambil dianalisis untuk mengekstraksi metadata, meliputi \textit{degree} (jumlah derajat antara \textit{source node} dan \textit{target node}) serta \textit{weight} (bobot kekuatan hubungan semantik), guna mengkuantifikasi signifikansi struktural dan kontekstualnya.

                        \item \textbf{\textit{Integration and Ranking}:} \textit{Edges} dikumpulkan dan diperingkat (\textit{ranked}) menggunakan metrik komposit yang menggabungkan skor relevansi (berdasarkan keselarasan kata kunci) dan bobot koneksi, untuk memastikan bahwa tautan konseptual yang dominan diprioritaskan.

                        \item \textbf{\textit{Text Unit Extraction}:} \textit{Text chunks} yang diasosiasikan dengan \textit{edges} diperingkat berdasarkan kemiripan antara \textit{edges} tersebut dengan kata kunci tingkat tinggi, kemudian diambil sebagai konteks pendukung.

                        \item \textbf{\textit{Data Truncation}:} Untuk mematuhi batasan komputasi, seluruh data yang diekstraksi, termasuk \textit{nodes}, \textit{edges}, dan \textit{text units}, dipangkas secara bertahap hingga total jumlah token memenuhi ambang batas yang telah ditentukan.

                    \end{enumerate}

                    \noindent
                    Mekanisme ini menghasilkan konteks global yang kaya akan pola hubungan, yang diformulasikan dalam persamaan:
                    \begin{equation}
                        \mathcal{L}_{\text{global}} = (\mathcal{E}_{\text{global}}, \mathcal{R}_{\text{global}}, \mathcal{T}_{\text{global}})
                        = \text{pipeline}_{\text{global}}(\mathcal{K}_h, G).
                    \end{equation}
                    \noindent
                    Kombinasi antara presisi tinggi dari \textit{Local Retrieval} dan wawasan luas dari \textit{Global Retrieval} inilah yang menjadi keunggulan arsitektur \textit{Graph-RAG} dalam menangani kompleksitas literatur ilmiah.

              \item \textbf{LLM Generation \& Response}

                    \hspace*{0.75cm}Tahap akhir mengirimkan augmented prompt ke \textit{LLM} yang berisi: (1) user query asli, (2) konteks terintegrasi dari tahap sebelumnya, dan (3) instruction untuk menghasilkan jawaban berdasarkan konteks. Karena konteks sudah melalui validasi dual-path (\textit{graph traversal} + \textit{vector similarity}), \textit{LLM} dapat menghasilkan jawaban yang lebih akurat dan terfokus dari \textit{knowledge graph}.
          \end{enumerate}

    \item \textbf{Evaluasi Sistem}
          \label{subsec:system-evaluation}

          \hspace*{0.75cm}Penelitian ini mengevaluasi kinerja sistem menggunakan standar \textit{RAGAS (Retrieval-Augmented Generation Assessment System)}, yang menyediakan metrik komprehensif untuk menilai dua dimensi utama: kemampuan pengambilan informasi relevan (\emph{retrieval quality}) dan kualitas jawaban (\emph{generation quality}).

          \hspace*{0.75cm}Skenario eksperimen membandingkan \textit{Graph-RAG} dan \textit{Vector-RAG} menggunakan 40 pertanyaan akademik yang mencakup aspek faktual, relasional, dan sintesis, dilengkapi \textit{ground truth} manual. Lima metrik utama digunakan sebagai tolok ukur:

          \begin{enumerate}[label=\arabic*., leftmargin=0.75cm, itemsep=3pt]

              \item \textbf{Context Precision}: Relevansi dokumen terhadap pertanyaan
                    \begin{equation}
                        \textit{Context Precision} = \frac{\sum_{k=1}^{K} \textit{Precision}(k) \cdot v_k}{\text{Total item relevan}}
                    \end{equation}

              \item \textbf{Context Recall}: Rasio ground truth yang didukung konteks
                    \begin{equation}
                        \textit{Context Recall} = \frac{\text{Kalimat GT didukung}}{\text{Total kalimat GT}}
                    \end{equation}

              \item \textbf{Answer Correctness}: Kecocokan jawaban dengan ground truth
                    \begin{equation}
                        \textit{Answer Correctness} = \frac{\text{TP}}{\text{TP} + 0.5 \cdot \text{FP} + \text{FN}}
                    \end{equation}

              \item \textbf{Faithfulness}: Rasio klaim terverifikasi (kontrol halusinasi)
                    \begin{equation}
                        \textit{Faithfulness} = \frac{V_S}{\text{V}_S}
                    \end{equation}

              \item \textbf{Answer Relevancy}: Keselarasan jawaban dengan pertanyaan
                    \begin{equation}
                        \textit{Answer Relevancy} = 1 - \frac{1}{n} \sum_{i=1}^{n} \text{sim}(q, q_i)
                    \end{equation}

          \end{enumerate}
          Penelitian menetapkan target keberhasilan sistem berdasarkan threshold berikut:
          \begin{itemize}[leftmargin=0.75cm]
              \item \textit{Context Precision} $> 0.85$ (relevansi tinggi)
              \item \textit{Context Recall} $> 0.80$ (cakupan informasi luas)
              \item \textit{Faithfulness} $> 0.90$ (minimal halusinasi)
              \item \textit{Answer Correctness} $> 0.75$ (keakuratan jawaban)
          \end{itemize}

          Sistem dianggap berhasil jika memenuhi standar pada seluruh metrik tersebut. Untuk validasi efektivitas, dilakukan perbandingan benchmarking antara Baseline (Standard Vector-RAG) dengan sistem Hybrid Graph-RAG yang diusulkan.


          % 6. GUI Implementation
    \item \textbf{Implementasi GUI (\textit{GUI Implementation})}

          \hspace*{0.75cm}Antarmuka pengguna (\textit{Graphical User Interface}) dikembangkan menggunakan \textit{Streamlit} untuk memfasilitasi interaksi pengguna. Sistem menampilkan jawaban, referensi sumber, dan visualisasi \textit{sub-graf} secara \textit{real-time} untuk memberikan transparansi kepada pengguna mengenai proses perolehan jawaban.
\end{enumerate}
